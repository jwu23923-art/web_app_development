# 個人記帳本系統 PRD

## 1. 專案概述

### 背景與動機
大學生常面臨生活費管理不當、月底「吃土」的問題。雖然市面上已有許多記帳 App，但有些過於複雜，或包含過多廣告。本案旨在開發一個介面簡潔、操作直覺的網頁記帳系統，幫助大學生輕鬆管理每日開支。

### 目標用戶
- 想要精確控管開支、養成儲蓄習慣的大學生。
- 需要簡單分類統計消費狀況的使用者。

### 核心價值主張
- **快速紀錄**：簡化輸入流程。
- **視覺化統計**：一眼看出錢花在哪裡。
- **跨裝置存取**：網頁版隨時隨地紀錄。

---

## 2. 功能需求

### 主要功能
1. **新增收支紀錄**：支援金額、類別（食物、交通、娛樂等）、備註與日期輸入。
2. **紀錄編輯與刪除**：若輸入錯誤可隨時修正或移除。
3. **即時餘額統計**：自動計算所有收支後的當前總額。
4. **分類比例圖表**：使用圖表呈現該月各類別的消費佔比。
5. **歷史紀錄清單**：按時間倒序排列所有歷史收支明細。

### 使用者故事
- 作為 **大學生**，我希望 **能快速輸入便當錢**，以便 **我能正確追蹤生活費剩餘額度**。
- 作為 **想要省錢的使用者**，我希望 **能查看上週在交通費上的支出比例**，以便 **調整下週的社交預算**。
- 作為 **管理開支的人**，我希望 **能修改寫錯的金額**，以便 **維持紀錄的準確性**。

---

## 3. 非功能需求

### 技術限制
- **開發框架**：Flask (Python)
- **模板引擎**：Jinja2 (HTML/CSS)
- **資料庫**：SQLite (輕量、單檔案)
- **CSS 樣式**：Vanilla CSS (現代、精美、動態設計)

### 效能與安全考量
- **響應式設計**：需支援手機瀏覽器。
- **資料完整性**：確保收支資料正確存入 SQLite，防止負數金額（非預期）或格式錯誤。
- **載入速度**：統計圖表載入不應造成明顯延遲。
