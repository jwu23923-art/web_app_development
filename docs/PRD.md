# PRD：個人記帳本 (Personal Expense Tracker)

## 1. 專案概述

### 背景與動機
大學生活中，許多學生面臨有限的生活費與多元的消費場景（餐飲、交通、娛樂、學習材料等），但卻缺乏一個簡單易用的工具來追蹤每日收支狀況。常見的問題包括：不知道錢花到哪裡去了、月底經常超支、難以建立儲蓄習慣。本系統希望提供一個輕量且直觀的網頁記帳工具，幫助大學生即時記錄收入與支出，並透過統計圖表掌握自身的消費模式，從而養成良好的理財習慣。

### 目標用戶
想要控管開支、掌握財務狀況的大學生。他們習慣使用手機或筆電上網，偏好簡潔直覺的操作介面，不需要複雜的財務分析功能，但希望能「一眼看清」自己的餘額與消費結構。

### 核心價值主張
提供一個零門檻、操作直覺的個人記帳網頁應用，讓使用者能在數秒內完成記帳，並即時獲得餘額與消費分類的視覺化回饋，助力大學生建立每日記帳的好習慣。

## 2. 功能需求

本系統主要功能包含以下七項：

### F1. 新增支出 / 收入紀錄 (Add Transaction)
- **使用者故事**：作為 **大學生使用者**，我希望 **能快速新增一筆收入或支出紀錄，填寫金額、類別、日期與備註**，以便 **即時記錄每一筆花費或收入，避免遺忘**。
- **欄位說明**：
  - 類型（收入 / 支出）
  - 金額（正整數或小數，最多兩位小數）
  - 類別（食物、交通、娛樂、學習、日用品、薪資、獎學金、其他）
  - 日期（預設為今天，可自行選擇）
  - 備註（選填，最多 100 字）

### F2. 編輯紀錄 (Edit Transaction)
- **使用者故事**：作為 **大學生使用者**，我希望 **能修改已建立的紀錄內容（如金額填錯、類別選錯）**，以便 **確保帳目資料的正確性**。

### F3. 刪除紀錄 (Delete Transaction)
- **使用者故事**：作為 **大學生使用者**，我希望 **能刪除不需要或誤建的紀錄**，以便 **維持帳本的整潔與準確**。
- **補充**：刪除前應顯示確認提示，避免誤刪。

### F4. 顯示目前總餘額 (Display Balance)
- **使用者故事**：作為 **大學生使用者**，我希望 **首頁就能看到目前的總餘額（總收入 − 總支出）**，以便 **隨時掌握可用資金**。
- **補充**：餘額以醒目的數字呈現於首頁頂部；當餘額為負數時以紅色警示。

### F5. 按類別查看消費比例 (Category Breakdown)
- **使用者故事**：作為 **大學生使用者**，我希望 **能透過圓餅圖或長條圖查看各類別的支出佔比**，以便 **了解自己的消費結構，找出最大花費來源**。
- **補充**：
  - 預設顯示「本月」的類別消費比例。
  - 可切換時間範圍（本週 / 本月 / 全部）。

### F6. 顯示歷史消費清單 (Transaction History)
- **使用者故事**：作為 **大學生使用者**，我希望 **能看到所有紀錄按日期排列的清單**，以便 **回顧過去的每一筆收支明細**。
- **補充**：
  - 清單預設以日期由新到舊排序。
  - 每筆紀錄顯示：日期、類型圖示、類別、備註、金額。
  - 收入以綠色顯示，支出以紅色顯示。

### F7. 按月份統計摘要 (Monthly Summary)
- **使用者故事**：作為 **大學生使用者**，我希望 **能查看每月的收入總計、支出總計與結餘**，以便 **追蹤自己每月的財務健康狀況**。

## 3. 非功能需求

### 技術限制
- **後端框架**：Flask
- **模板引擎**：Jinja2
- **資料庫**：SQLite
- **前端**：HTML5 + CSS3 + JavaScript（可搭配 Chart.js 繪製圖表）
- **不使用外部認證系統**：本版本為單人使用，不實作登入功能

### 效能與安全考量
- **響應式設計**：系統應在手機與桌面瀏覽器上都能流暢操作（RWD）。
- **資料驗證**：所有表單提交均需進行伺服器端驗證（金額必須 > 0、日期格式正確、類別必須為預設選項之一）。
- **SQL 注入防護**：所有資料庫操作使用參數化查詢，防止 SQL Injection。
- **XSS 防護**：使用者輸入的備註欄位需進行轉義處理。
- **資料備份**：SQLite 檔案應可手動複製備份。

### 使用者體驗
- 新增紀錄流程不超過 3 個步驟。
- 頁面載入時間不超過 2 秒。
- 操作回饋即時明確（新增成功、刪除成功等提示訊息）。

## 4. MVP 範圍

### Must Have（必須有）
- 新增收入 / 支出紀錄（F1）
- 刪除紀錄（F3）
- 顯示目前總餘額（F4）
- 顯示歷史消費清單（F6）

### Should Have（應該有）
- 編輯紀錄（F2）
- 按類別查看消費比例圖表（F5）
- 刪除前確認提示

### Nice to Have（加分項）
- 按月份統計摘要（F7）
- 時間範圍篩選（本週 / 本月 / 全部）
- 匯出紀錄為 CSV
- 深色模式切換
