# PRD：個人食譜收藏夾 (Personal Recipe Collection)

## 1. 專案概述
- **背景與動機**：愛好下廚的人常會從各處（網站、雜誌、長輩親友）收集食譜，但這些資訊往往分散且難以整理。本系統希望提供一個專屬的空間，讓使用者能記錄、分類並管理自己喜愛的私藏食譜，成為個人專屬的料理筆記。
- **目標用戶**：喜歡下廚、希望整理個人食譜清單、對料理分類有需求的居家料理者。
- **核心價值主張**：簡單、直觀且結構化的個人食譜管理工具，幫助使用者快速尋找與回顧料理靈感。

## 2. 功能需求
本系統主要功能包含以下五項：

1. **新增食譜 (Add Recipe)**
   - 作為 **料理使用者**，我希望 **能記錄菜名、食材清單與烹飪步驟**，以便 **將學到的新料理存檔備查**。
2. **食譜分類標籤 (Category Tags)**
   - 作為 **料理使用者**，我希望 **能依照菜系（如：台式、日式、甜點）為食譜打上標籤**，以便 **對收藏進行有系統的分類**。
3. **顯示食譜清單 (Recipe List)**
   - 作為 **料理使用者**，我希望 **能看到所有已收藏食譜的總覽清單**，以便 **快速瀏覽我所有的料理收藏**。
4. **編輯與刪除 (Edit & Delete)**
   - 作為 **料理使用者**，我希望 **能調整舊有的食譜內容或刪除不再需要的食譜**，以便 **隨時修正做法或精簡收藏清單**。
5. **分類篩選功能 (Filter by Category)**
   - 作為 **料理使用者**，我希望 **能依照分類標籤快速篩選食譜**，以便 **在想煮特定類別料理時能迅速找到目標**。

## 3. 非功能需求
- **技術限制**：
  - 後端框架：Flask
  - 模板引擎：Jinja2
  - 資料庫：SQLite
- **效能與安全考量**：
  - 系統應能在各行動裝置與案頭端流暢運行（響應式設計基礎）。
  - 資料庫設計需確保食材清單與步驟的完整性，不論文字長短都能正確顯示。
  - 搜尋與篩選應具備即時性，確保使用者在廚房忙碌時能快速取得資訊。

## 4. MVP 範圍
- **Must Have (必須有)**：
  - 食譜的新增、讀取（列表）、刪除功能。
  - 基礎欄位：菜名、食材、步驟。
- **Should Have (應該有)**：
  - 食譜的編輯修改功能。
  - 基本的分類標籤功能與分類篩選。
- **Nice to Have (加分項)**：
  - 烹飪時間、難易度標記。
  - 食譜圖片預覽（可透過輸入圖片連結實現）。
  - 關鍵字搜尋功能。
